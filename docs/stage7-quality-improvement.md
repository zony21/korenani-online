# 第7段階 品質向上

## 目的

第5段階・第6段階でゲームの基本進行と終了後導線が入ったため、第7段階では遊びやすさと安定性を上げる。

## 対象

1. 結果画面の情報強化
2. 質問履歴の見やすさ向上
3. 入力バリデーション強化
4. 二重送信防止
5. 再接続時の状態復元
6. エラー表示の改善

## 実装方針

大きな画面ファイルを一括更新すると反映が不安定になるため、小さい単位で反映する。

### 7-1 結果画面の情報強化

ゲーム終了時に以下を表示する。

- 勝者
- 正解のお題
- 到達ターン
- 質問数
- 解答数
- 回答数

### 7-2 質問履歴の見やすさ向上

質問履歴に以下を表示する。

- ターン番号
- 質問者
- 質問内容
- 回答結果

### 7-3 入力バリデーション強化

以下を禁止する。

- 空白のみ
- 長すぎる名前
- 長すぎるテーマ
- 長すぎるお題
- 長すぎる質問
- 長すぎる解答

### 7-4 二重送信防止

送信中はボタンを非活性にする。

### 7-5 再接続時の状態復元

画面更新後も以下を復元する。

- 自分のプレイヤーID
- 現在のルーム状態
- 現在のフェーズ
- 回答済み状態

## 次の実装順

1. バリデーション定数追加
2. フロント入力上限追加
3. バックエンド入力上限追加
4. 結果サマリー部品追加
5. GameViewへ小分け反映
