# これなに？オンライン

リアルタイムで遊ぶ推理ゲームです。
各プレイヤーがゲーム開始前に「答え」を1つ提出し、開始時に自分以外の誰かが提出した答えがランダムに割り当てられます。
プレイヤーは自分に割り当てられた答えだけ見えない状態で、質問と解答を使って自分の答えを推理します。

## ゲーム基本仕様

### 人数

```text
2〜9人
```

1人では開始できません。

### ルーム作成

入力項目:

- 名前
- テーマ
- ターン数
- パスワード有無
- パスワードありの場合は5桁英数字

ターン数:

- 10ターン
- 20ターン
- 30ターン

### 答え提出

ゲーム開始前に、各プレイヤーが答えを1つ提出します。

例:

```text
A → りんご
B → 犬
C → 車
```

この時点では、まだ誰にどの答えが割り当たるかは決まっていません。

### 答え割当

ゲーム開始時、提出された答えをプレイヤーへランダム配布します。
ただし、自分が提出した答えは絶対に自分には割り当てません。

例:

```text
A提出 → りんご
B提出 → 犬
C提出 → 車

A ← 犬
B ← 車
C ← りんご
```

### 表示ルール

隠す対象は「自分が提出した答え」ではなく、「自分に割り当てられた答え」です。

例:

```text
A本人の画面
A：？？？
B：車
C：りんご

B本人の画面
A：犬
B：？？？
C：りんご
```

実装上の基準は `assignedTopic` です。
提出者情報は表示判定に使いません。

## ターン進行

現在手番プレイヤーのみ、以下を選択できます。

1. 質問する
2. 解答する

### 質問

手番プレイヤーが質問を入力します。
他プレイヤーは、そのプレイヤーに割り当てられた答えを見て回答します。

回答選択肢:

- はい
- いいえ
- どちらともいえない

回答条件:

- 全員回答
- または3分経過

回答結果は1分表示され、その後次ターンに進みます。

### 解答

入力した解答と、自分の `assignedTopic` を比較します。

- 一致: 勝利してゲーム終了
- 不一致: 次プレイヤーへ進行

## ゲーム終了

終了条件:

- 誰かが正解
- ターン上限到達

終了後操作:

- 同じテーマで次のゲーム
- テーマを変えて次のゲーム
- 部屋を閉じる

## 重要な実装方針

- `main` が唯一の正
- 過去ブランチは参照しない
- 大型ファイルは差分修正優先
- `GameView.vue` は大きいため特に小分けで修正する
- DB変更は慎重に行う
- Prismaスキーマ変更時は `npx prisma db push` を実行する
